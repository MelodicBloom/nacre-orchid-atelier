# NACRE ORCHID ATELIER

> Living material design system — tokens, GLSL shaders, React MaterialSurface, and Next.js docs atlas. Powered by the [Nacre Inquiry Engine](https://github.com/MelodicBloom/nacre-inquiry-engine).

---

## What This Is

NACRE ORCHID ATELIER is a fully deterministic, inquiry-driven design system where **every token, shader, and component traces back to a single semantic profile** generated by the Nacre Inquiry Engine. The visual language is built from living materials: pearl iridescence, orchid petal gradients, lace micropatterns, abalone thin-film, opal diffraction, labradorite schiller, and gold cloisonné line work.

## Repository Architecture

```
nacre-orchid-atelier/
├── packages/
│   ├── tokens/          # Design token pipeline (JSON → CSS/JS/Tailwind)
│   ├── shaders/         # GLSL iridescent surface library (extends aether)
│   ├── components/      # React MaterialSurface component library
│   └── docs/            # Next.js documentation atlas
├── .ai/
│   └── prompts/         # Governed AI coding, review, and docs prompts
├── .github/
│   └── workflows/       # CI/CD — type check, lint, visual regression, build
├── SPEC.md              # Master build specification
├── GATES.md             # Stage gate checklist
└── QA.md                # Quality assurance protocol
```

## Quick Start

```bash
npm install

```

## Product Loop

```
Nacre Inquiry Engine  →  semantic profile JSON
       ↓
  Token Pipeline      →  CSS vars / Tailwind / JS tokens
       ↓
  Aether Shaders      →  iridescent GLSL surface layer
       ↓
  MaterialSurface     →  React components consuming tokens + shaders
       ↓
  Docs Atlas          →  Next.js living documentation site
```

## Governance

All changes must pass the **5 Stage Gates** defined in [`GATES.md`](./GATES.md) before merging to `main`. AI-assisted sessions must use the prompts in [`.ai/prompts/`](./.ai/prompts/).
